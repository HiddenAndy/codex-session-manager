# Patch Notes Guide

업데이트 모달에 보여줄 패치노트 작성 기준입니다.

## 원칙

- 사용자가 바로 이해할 수 있는 변화 위주로 씁니다.
- 내부 구현명은 꼭 필요할 때만 씁니다.
- 한 항목은 한 줄에 끝나도록 짧게 씁니다.
- 과하게 친근한 말투나 반말은 쓰지 않습니다.
- 제목은 모달 타이틀에 버전이 나오므로 본문에는 기능 묶음 제목만 둡니다.

## 지원하는 마크다운

업데이트 모달은 제한된 마크다운만 렌더링합니다.

- `##`, `###`, `####` 제목
- `-` 목록
- `**굵게**`
- `` `인라인 코드` ``

표, 링크, 이미지, 번호 목록, 중첩 목록은 쓰지 않습니다.

## 권장 구조

```md
## 이번 업데이트

이번 버전에서 정리한 내용을 한 문장으로 적습니다.

### 주요 변경
- **핵심 기능명**을 사용자가 이해하기 쉬운 말로 설명합니다.
- 문제를 고친 경우 무엇이 더 이상 발생하지 않는지 씁니다.

### 사용성 개선
- 화면에서 바로 느껴지는 변화나 조작 흐름 개선을 씁니다.
- 기준이나 표시가 바뀐 경우 이전 표현과 새 표현을 같이 씁니다.
```

## 0.1.4 예시

```md
## 이번 업데이트

헷갈리던 표시와 삭제 후 다시 보이던 문제들을 정리했습니다.

### 정리한 것
- **프로젝트 경로 변경/재설정** 흐름을 안정화했습니다.
- 삭제한 채팅이 다시 보이던 문제를 수정했습니다.
- 백업/복원 후 남는 깨진 DB 기록을 정리하도록 했습니다.
- 상단 문구를 **채팅/프로젝트/백업/진단 요약**으로 바꿨습니다.

### 사용성 개선
- 검색어를 입력하면 결과 프로젝트가 자동으로 펼쳐집니다.
- **압축 고려** 뱃지는 제거하고 새 채팅 권장 기준만 남겼습니다.
- 새 채팅 권장 뱃지에 설명 툴팁을 추가했습니다.
- 보조 패널은 화면을 넘지 않고 내부에서 스크롤됩니다.
- 사용 가이드에 작은 **이스터에그**를 넣었습니다.
```
